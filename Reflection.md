This assignment provided an opportunity to enhance my knowledge of secure development for web applications by using three different technologies (SQLAlchemy, Pydantic, and password hashing) together to provide a model for managing users securely. Creating a separation of concerns between models used to represent the database and schema used for validation allowed me to see how all of the data passes through the Fast API application based on good security practices. By implementing a password hash in the code, I was able to see firsthand why you cannot store any sensitive information in plain text format.

The most difficult portion of this assignment involved getting PostgreSQL, Docker, GitHub Actions, and Docker Hub working together. A small configuration error could cause the entire process to fail. Thus, troubleshooting connectivity issues with databases, configurations for containers, and deployment were necessary. In addition, developing unit and integration tests assisted me in realizing how important it is to test both the performance of single functions within an application as well as the overall function of the application itself to increase software reliability.

Utilizing Git and GitHub Actions showed the benefit of committing regularly and having testing occur during automation. Testing occurred every time there was a commit, which gave me more confidence that any additional code added would not have caused any regression. Additionally, using Docker made it much easier to create packages of the application to ensure consistency across multiple environments. Overall, this assignment enhanced my knowledge of secure user authentication, automated testing, database integration, containerization, and continuous integration. As a result of completing this assignment, I am aware of many areas in which I wish to further develop my DevOps and deployment capabilities.
