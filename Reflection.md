Module 12 provided significant learning experiences through which I further developed my calculator application utilizing FastAPI to include user authentication along with full CRUD functionality (BREAD) for users performing calculations. Additionally, this module enhanced my ability to understand the operation of REST APIs and how they relate HTTP requests to database operations, as well as the creation of SQLAlchemy models that utilize Pydantic validation. Furthermore, I have been able to better understand how JWT authentication can be utilized to protect API endpoints and how a user's registration and login are performed within a web-based application. Lastly, testing each endpoint via Swagger has greatly assisted me in validating that each endpoint functioned appropriately prior to executing the automated test suite.

The most difficult challenge I experienced during this module was identifying and resolving several database initialization and API routing issues. Initially, some endpoints did not function due to database table errors; therefore, I had to analyze each error message to determine the exact problem. Utilizing unit, integration, and end-to-end testing methods proved beneficial in verifying that each problem identified had been successfully addressed without introducing any additional bugs. Further, the use of GitHub Actions facilitated rapid detection of problems by allowing the execution of the automated test suite after pushing changes to GitHub.

This module demonstrated the value of using Git, automated testing and CI/CD throughout the entire development process. Through making frequent commits, I was able to monitor my development progress and more effectively recover from any errors or mistakes encountered. Overall, I believe that I have obtained very relevant experience developing secure REST APIs utilizing FastAPI and PostgreSQL. I am now significantly more confident developing larger-scale applications.
